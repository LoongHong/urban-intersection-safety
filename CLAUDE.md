## Project Overview

Final Year Project (FYP): Traffic accident **risk level classification** for intersection-related crashes in Chicago. The goal is to predict a 3-class `Risk_Level` target (0 = Low, 1 = Medium, 2 = High) from environmental and temporal features.

**Dataset:** `traffic_accidents.csv` — 209,306 rows, 24 columns. Path is hardcoded in `datapreprocessing.ipynb`:

## Notebook Workflow

Two notebooks run sequentially:

### 1. `datapreprocessing.ipynb`
- Filters to intersection-only crashes (`intersection_related_i == 'Y'`) → ~163,676 rows
- Drops rows with `UNKNOWN` / `UNABLE TO DETERMINE` in categorical columns (not imputed)
- Assigns `Risk_Level` via a rule-based function:
  - **High (2):** `injuries_fatal > 0` or `most_severe_injury` is FATAL/INCAPACITATING
  - **Medium (1):** NONINCAPACITATING or REPORTED NOT EVIDENT injury, or OVER $1,500 damage with injuries
  - **Low (0):** everything else
- Runs Chi-Square tests (categorical features) and One-Way ANOVA (numerical features) for feature validation
- Creates two train/test splits (80/20 and 70/30), both stratified
- Applies **SMOTE** to training sets only to balance all three classes to 33.3% each
- Two preprocessor variants are fit and saved:
  - `preprocessor_lr`: StandardScaler + OneHotEncoder (for Logistic Regression, produces 67 features)
  - `preprocessor_tree`: StandardScaler + OrdinalEncoder (for tree models, keeps 10 features)
- Saves all encoded arrays and preprocessors to **`fyp_preprocessed_data.pkl`** via `joblib`

### 2. `modeltraining.ipynb`
- Loads `fyp_preprocessed_data.pkl`
- Trains **Random Forest** and **XGBoost** on both splits (tree preprocessor arrays)
- `evaluate_model()` helper prints accuracy, F1-macro/weighted, ROC-AUC, and saves confusion matrix PNGs named `cm_{ModelName}_{split}.png`
- Hyperparameter tuning via `RandomizedSearchCV` (15 iterations, 3-fold CV, scoring=`f1_macro`)
- All results collected in a `results` list for final comparison

## Running the Notebooks

Open and run cells top-to-bottom in order:
```
datapreprocessing.ipynb  →  generates fyp_preprocessed_data.pkl
modeltraining.ipynb      →  loads pkl, trains and evaluates models
```

Cells in `modeltraining.ipynb` have an import bug: `RandomizedSearchCV` import inside the RF tuning cell does not persist to the XGBoost tuning cell — add the import at the top of the XGBoost tuning cell if running sequentially.
